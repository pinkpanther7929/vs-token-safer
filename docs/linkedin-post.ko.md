코딩 에이전트한테 큰 저장소를 맡겨 보면 늘 같은 데서 막힙니다. "이 함수 어디 있어?" 한마디에 grep이 수천 줄을 긁어다 컨텍스트 창에 쏟아붓고, 정작 중요한 맥락은 그 더미에 밀려 사라집니다.

그래서 vs-token-safer를 만들었습니다. 에이전트와 저장소 사이에서, 소스를 통째로 퍼오는 대신 짧은 file:line 목록 하나로 답해 주는 작은 레이어입니다. Claude Code 플러그인이자 MCP 서버이고, 전부 로컬에서 돕니다. 아무것도 밖으로 나가지 않습니다.

핵심은 하나의 "정밀도 사다리"입니다. 아는 만큼 정확하게 답하고, 모르면 한 단 내려가 폴백합니다:

🎯 이름을 알면 → 공식 언어 서버(clangd · Roslyn · tsserver · pyright)가 시맨틱으로 정확히
🌳 툴체인이 없으면 → tree-sitter가 36개 언어를, 설치 없이 바로
🔍 이름은 잊고 "auth 흐름이 어떻게 되더라"만 떠오르면 → 저장소가 직접 쓰는 단어로 찾아냅니다 (임베딩 없음)
📄 코드가 아니라 문서·설정이면 → Markdown·YAML·TOML을 헤딩으로 바로

게다가 한 단에 머물지 않습니다. 이름을 몰라 fuzzy로 출발해도, 후보 이름이 잡히는 순간 그대로 시맨틱 단으로 올라타 정답을 확정합니다. 답이 어느 단에서 나왔는지까지 한 줄로 알려주니, 에이전트가 시맨틱 정답을 받았는지 폴백을 받았는지 헷갈릴 일이 없습니다.

숫자로 보면:
• 3개 언어·150파일 벤치마크에서 grep 대비 토큰 87% 절감
• 실제 Unreal Engine 5 트리(26,000여 번역 단위)에서 심볼 하나 찾기 약 138배 절감

여기에 한 가지를 더 했습니다. Claude가 자꾸 grep으로 손이 가면, 훅이 그 명령을 그 자리에서 인덱스 질의로 바꿔치기합니다. 막아서 흐름을 끊는 게 아니라, 검색은 그대로 돌아가게 두면서 더 싼 경로로 돌려놓는 거죠.

게임 엔진처럼 무거운 C++ 트리에서 출발했지만, 지금은 TypeScript·JavaScript·Python·C#·C++·Go까지 같은 방식으로 다룹니다. 웹이든 백엔드든, 큰 저장소를 에이전트에게 맡기는 분이라면 바로 쓰실 수 있습니다.
